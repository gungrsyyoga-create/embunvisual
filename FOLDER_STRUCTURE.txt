FOLDER GENERATION FLOW:
═══════════════════════════════════════════════════════════════

When Super Admin creates folder for client:

1. INPUT:
   - Invoice: INV-2026-001
   - Klien: "PT Event Organizer"
   - Tier: "premium"

2. FOLDER CREATED:
   /undangan/premium/pt-event-organizer/
   
3. TEMPLATE COPIED:
   tema_premium_rsvp.php → /undangan/premium/pt-event-organizer/index.php

4. DATABASE UPDATED:
   klien_premium.tipe = "premium"
   klien_premium.folder_path = "undangan/premium/pt-event-organizer"
   klien_premium.is_active = 1

5. AUDIT LOGGED:
   audit_logs entry with:
   - action: "Generate Folder Client"
   - keterangan: "Invoice: INV-2026-001, Tier: premium, Path: ..."

6. RESULT:
   URL: /undangan/premium/pt-event-organizer/index.php?pid=X
   Guest dapat akses invitation dengan:
   - 🎉 Hero section dengan logo
   - ⏳ Countdown timer (real-time)
   - 📝 RSVP form
   - 📊 Live statistics (Hadir/Tidak Hadir)


DATABASE SCHEMA UPDATES:
═══════════════════════════════════════════════════════════════

✓ klien_premium table:
  ├── tipe ENUM('basic','premium','exclusive')
  ├── folder_path VARCHAR(255)
  └── is_active TINYINT(1)

✓ admin_users table:
  └── role ENUM('Super Admin','Staff','Basic')

✓ barcode_scans table (NEW):
  ├── id INT PRIMARY KEY AUTO_INCREMENT
  ├── pesanan_id INT FOREIGN KEY
  ├── barcode VARCHAR(100) UNIQUE
  ├── nama_tamu VARCHAR(200)
  ├── scan_time TIMESTAMP
  └── scan_ip VARCHAR(45)

✓ tamu_undangan table (existing):
  └── Used for RSVP tracking
